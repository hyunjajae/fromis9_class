# 화상수업 위장 유튜브 플레이어 (1단계: 웹)

구글미트 갤러리 뷰처럼 생긴 가짜 화상수업 화면입니다. 참가자 타일 중 한 칸에서
유튜브 영상을 재생하고, **ESC**를 누르면 그 칸이 "카메라 끈 타일"로 바뀌며 음소거되어
빠르게 위장할 수 있습니다.

- 단일 `index.html` + 바닐라 JS, 외부 빌드 없음
- 설정은 메모리 상태로만 관리 (localStorage 등 스토리지 미사용)
- 정적 호스팅으로 배포 가능

## 실행 방법

카메라(`getUserMedia`)는 **보안 컨텍스트(https 또는 localhost)** 에서만 동작합니다.
`index.html`을 파일로 직접 더블클릭(`file://`)하면 웹캠이 켜지지 않으니, 로컬에서는
아래처럼 **localhost로 서빙**하세요.

```bash
# Python 3
cd fake-class-player
python -m http.server 8000
# → http://localhost:8000/ 접속

# 또는 Node
npx serve .
```

설정 화면에서 입력:

1. **수업/회의 이름** (비우면 "온라인 수업")
2. **그리드 크기** 2 / 3 / 4 / 5 (= 2×2 ~ 5×5). 그 외 값은 막힘.
3. **유튜브 링크** (`youtu.be/...`, `watch?v=...`, `embed/...`, `shorts/...`, `live/...`,
   또는 11자리 ID 모두 인식)

"시작" → 메인 화상수업 화면.

## 사용법

- **ESC**: 패닉 토글. 프레젠터 칸(유튜브)만 즉시 정지+음소거되며 화면 끔 타일로 바뀝니다.
  다시 누르면 복귀. (웹캠은 ESC와 무관 — 아래 카메라 버튼으로 별도 제어.)
- **하단 카메라 버튼**: 내 웹캠("나" 칸)을 실제로 켜고 끕니다(`track.enabled` 토글).
  끄면 화면 끔 타일, 다시 켜면 영상 표시. 카메라 권한이 없으면 동작하지 않습니다.
- **하단 마이크 버튼**: 오디오 기능은 없고 표시(빨간 사선)만 토글됩니다.
- **영상 바꾸기**(상단바 버튼): 새 유튜브 링크를 붙여넣어 즉시 교체.

## 타일 배치 규칙

전체 칸 = N×N. 프레젠터 1 + 나 1 고정, 나머지를 프로필/화면끔으로 분배합니다.
**프로필 중 1칸은 항상 "교수님"** 이며(그리드 크기 무관 무조건 포함), 나머지 프로필은 학생 사진입니다.

| 그리드 | 칸 | 프레젠터 | 나 | 프로필(교수님+학생) | 화면끔 |
|--------|----|----------|----|--------|--------|
| 2×2 | 4 | 1 | 1 | 1 (교수님1 + 학생0) | 1 |
| 3×3 | 9 | 1 | 1 | 4 (교수님1 + 학생3) | 3 |
| 4×4 | 16 | 1 | 1 | 7 (교수님1 + 학생6) | 7 |
| 5×5 | 25 | 1 | 1 | 12 (교수님1 + 학생11) | 11 |

> 분배 예시에 맞춰 **나머지의 절반은 프로필(ceil), 나머지는 화면끔(floor)** 으로
> 계산합니다(코드 `generateLayout`). 프레젠터/나 위치는 매번 섞이며 가능하면 모서리는
> 피합니다.

## 준비물 연결 지점 (추후 교체)

- **참가자 이름(이름표)**: `index.html` 상단 `FAKE_NAMES` 배열 (더미 26개, 5×5에서도 안 겹침).
  추후 실제 명단으로 교체하세요. (사진과 별개로 이름표 용도입니다.)
- **학생 사진**: `./assets/profiles/frame_001.jpg ~ frame_024.jpg` (24장).
  확장자는 jpg → png → jpeg 순으로 자동 인식. 개수/확장자는 `STUDENT_IMAGE_COUNT`,
  `STUDENT_IMAGE_EXTS` 로 조정.
- **교수님 사진**: `./assets/profiles/교수님.png` (또는 `교수님.jpg`). 항상 한 칸에 표시됩니다.
- 파일이 없거나 로드 실패 시 **색 배경 + 이니셜** 플레이스홀더로 폴백합니다.
  → `assets/profiles/README.txt` 참고.

## 2단계 (Electron) 대비 구조

Electron 래핑 시 그대로 재사용할 수 있도록 모듈/함수가 분리되어 있습니다.

- `KeyHandler` — 키 입력 핸들러. 웹은 `document` keydown으로 ESC 바인딩.
  2단계에서 `globalShortcut.register('Escape', togglePanic)` 등으로 교체.
- `YouTubeController` — 유튜브 생성/재생/정지/음소거/교체/ID 파싱.
- `WebcamController` — 웹캠 시작/`track.enabled` 토글/정지.
- `togglePanic()` / `applyDisguise(on)` — 위장 토글 핵심 로직(유튜브 칸만 제어).
- `toggleCamera()` / `setCameraOn(on)` — 웹캠 on/off (카메라 버튼 전용, ESC와 분리).
- 외부 호출용으로 `window.FakeClassPlayer`에 위 모듈과 `togglePanic`, `toggleCamera`,
  `setVideo(url)` 노출.
